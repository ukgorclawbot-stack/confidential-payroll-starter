# Zama Builder Application Draft

This document collects ready-to-use application text for submitting the project to the Zama Developer Program.

## Primary application version

```text
Application Title:
Confidential Payroll Starter Pack for Zama

Track:
Builder Track

Name:
[Your real name or handle]

Short Introduction:
I am a Web3 contributor focused on turning complex infrastructure into practical builder resources, documentation, and early-stage product concepts. I am especially interested in real-world use cases where confidentiality is essential for adoption.

What do you want to build?
I am building a documentation-first starter project around confidential payroll, one of the clearest real-world applications for confidential blockchain infrastructure. The project is designed to help builders understand how payroll execution can happen onchain without exposing sensitive compensation data publicly.

Why this project?
Most onchain systems are transparent by default, which makes them difficult to use for business-facing financial workflows such as payroll, bonuses, reimbursements, and contractor payments. Confidential payroll is a strong use case because it highlights the need for both programmability and privacy. I believe this makes it a useful entry point for other builders exploring Zama.

What will you deliver?
I plan to deliver:
1. A concept note explaining why confidential payroll matters onchain
2. A builder-friendly tutorial describing the use case and system design
3. A minimum architecture document covering actors, privacy boundaries, and workflow
4. A sample payroll flow showing how the system could operate in practice
5. A foundation for a future minimal prototype

Why is this useful to the ecosystem?
This contribution helps make confidential applications more concrete for new builders. Instead of presenting confidentiality as an abstract capability, it frames it as a practical workflow with clear user and business value. It can also serve as a reusable starting point for related applications such as bonuses, reimbursements, contractor payouts, and confidential treasury operations.

Current stage:
Documentation-first starter repository, with implementation direction prepared for a future prototype.

Long-term direction:
If the initial contribution is useful, I would like to extend it into a more concrete prototype and explore additional confidential finance workflows on top of Zama.

Links:
GitHub: [your repo link]
X: [your X profile]
Other relevant work: [optional]
```
